Safety Dataset Overview

The RAILS safety dataset contains reasoning traces where every intermediate step has been symbolically verified. Generated by a teacher model (DeepSeek) and validated by domain-appropriate solvers, these traces are used for SFT warmup before GRPO training.

Verification Backends

SAT Solver

Propositional logic. Tests unsatisfiability of premises ∪ ¬conclusion. Binary entailment check via MiniSAT.

ASP (Answer Set Programming)

First-order and non-monotonic logic. Cautious reasoning over stable models via Clingo. Handles disjunctions, classical negation, negation-as-failure, exceptions, and priorities.

NSFR (Neural-Symbolic Forward Reasoner)

First-order logic (supplementary). Forward chaining over Prolog-style rules and facts. Checks entailment via symbolic TP operator.

Generation Pipeline

Teacher LLM → Parse <step> tags → Format check → Symbolic verification → Structural filters → Answer match → Safety dataset

Each generated trace passes through multi-gate validation. The consistency gate dispatches to the domain-appropriate solver: SAT for propositional, ASP/Clingo for first-order and non-monotonic logic. Traces failing any blocking gate are re-generated (up to 3 retries). The same solvers are reused during GRPO training as reward functions.
