Why v8?

The v7 dataset contained only LogicBench examples. Models trained on v7 achieved strong LogicBench performance but failed catastrophically on other benchmarks because they never saw those formats during training:

BenchmarkBase QwenL1 (v7 data)GapRoot Cause
LogicBench83.6%61.8%-21.8pp"Unknown" bias from 3-class mixing
ProntoQA98.7%73.3%-25.4ppZero ProntoQA-style examples in training
ProofWriter80.7%40.0%-40.7ppZero ProofWriter examples in training
FOLIO~70%~45%-25ppZero FOLIO-style examples in training
Never use v7 for GRPO training. The file proof_bank_v7_grpo_v3.jsonl (67K) contains only LogicBench and produces models that cannot generalize. Always use proof_bank_v8_diverse_grpo.jsonl (88K).

Dataset Sources

LogicBench (Core)

67,449 examples (76.3%)

Programmatic proof chains from Proof Bank v7. Balanced across propositional (8 theories), FOL (9 theories), and non-monotonic (8 theories). 100% structurally verified. This is the same data as v7.

LogicBench-Aug NEW

12,908 examples (14.6%)

Augmented LogicBench examples with varied phrasings, entity names, and rule combinations. Increases diversity within the LogicBench format to prevent overfitting to specific patterns.

ProofWriter NEW

5,000 examples (5.7%)

Multi-hop propositional reasoning with depth 0-5. Three-class answers (True/False/Unknown). Uses "if...then" natural language rules about entities with properties. Critical for CWA reasoning.

FOLIO-style NEW

2,000 examples (2.3%)

Natural language premises with FOL and non-monotonic reasoning. Real-world scenarios (theater troupes, aviation centers, apiaries). Three-class answers (True/False/Uncertain). Covers complex quantified reasoning.

ProntoQA-style NEW

1,000 examples (1.1%)

Predicate-chain reasoning with invented entity types (e.g., "harmonious(vineyard) if ~precise(vineyard)"). Multi-hop FOL chains requiring forward chaining through rules. Binary yes/no answers.

Concrete Examples

ProofWriter (depth-based propositional reasoning)

ProofWriter: Unknown (CWA reasoning) propositional Unknown
Anne is cold. Anne is furry. Harry is rough. Rough things are smart. Question: Anne is smart.

The model must recognize that no rule connects Anne's properties (cold, furry) to "smart." Since rough→smart exists but Anne is not rough, and no other path exists, the answer is Unknown by CWA. This is exactly the type of reasoning the v7-trained model failed at (4/6 unknown labels wrong).

ProofWriter: True (multi-hop chain) propositional True
Gary is big. Gary is round. Gary is white. White things are young. Big, kind things are green. Question: Gary is young.

Chain: Gary is white → white things are young → Gary is young. A depth-1 inference the model must follow correctly without inventing additional rules.

FOLIO-style (natural language FOL premises)

FOLIO-style: False (complex quantified reasoning) FOL False
Premises: At the aviation training center, Hollis is noted for being agile. Emma is recognized as enthusiastic. Whenever someone is perceptive and compassionate, they naturally become inventive. If an individual is enthusiastic, they are inevitably creative. A creative person is always generous. Agility in a person leads to them being perceptive. Being inventive ensures that a person is also eloquent. Question: Hollis is not perceptive.

Chain: Hollis is agile → agile implies perceptive → Hollis IS perceptive → "not perceptive" is False. Uses natural language premises rather than formal Prolog-style rules, matching FOLIO's evaluation format.

FOLIO-style: False (non-monotonic with exceptions) non-monotonic False
Premises: In the apiary, Hazel is known to be ambitious, while Luca also has the trait of being ambitious. Someone who is ambitious is generally considered methodical, unless there is evidence to the contrary. A methodical individual is typically organized. If someone is meticulous, they are usually recognized as wise... Question: Hazel is not methodical.

Non-monotonic reasoning: Hazel is ambitious, and ambitious people are "generally" methodical "unless there is evidence to the contrary." With no contrary evidence, Hazel IS methodical, making the negation False. Requires default reasoning with exception handling.

ProntoQA-style (predicate chains)

ProntoQA-style: Yes (multi-hop forward chaining) FOL Yes
ambitious(elliot) if generous(elliot). generous(elliot) if polite(elliot). motivated(elliot) if insightful(elliot). polite(elliot). dedicated(elliot) if ambitious(elliot). Prove: ambitious(elliot)

Chain: polite(elliot) [fact] → generous(elliot) [rule 2] → ambitious(elliot) [rule 1]. A 2-hop chain matching ProntoQA's evaluation format. The model must follow the chain forward without reversing implications.

ProntoQA-style: No (non-monotonic with exceptions) non-monotonic No
ready_to_compete(player_leon) if driven(player_leon). communicative(player_leon) if focused(player_leon). focused(player_leon) if ready_to_compete(player_leon). focused(player_leon). exception: ~driven(player_leon) if communicative(player_leon). Prove: ready_to_compete(player_leon)

Non-monotonic chain: focused → communicative (via focused → ready_to_compete → driven, but exception: communicative blocks driven). The exception prevents the proof. The model must handle negation-as-failure correctly.

LogicBench-Aug (augmented inference rules)

LogicBench-Aug: No (disjunctive syllogism) propositional No
If Levi failed the test, then he will not get a reward. If he will get a reward, does this imply that Levi failed the test?

Tests understanding of implication directionality. failed → ~reward does NOT mean reward → failed (affirming the consequent fallacy). This is exactly the error pattern (reversed implications) that caused 8/22 failures in the L1 evaluation.

LogicBench (Core): True (existential reasoning) FOL True
In the ballet company, principal_neil, soloist_riley, and ballet_master_vera are part of the group. principal_neil possesses the quality of being methodical. Everyone who is methodical is considered thoughtful. Question: There exists someone in the ballet company who is thoughtful.

Existential quantification: principal_neil is methodical → methodical implies thoughtful → principal_neil is thoughtful → there exists someone who is thoughtful. Core FOL reasoning from the LogicBench base.
