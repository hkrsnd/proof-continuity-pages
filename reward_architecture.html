Reward Components

🔍 Consistency UPDATED

Per-step SAT/ASP verification. Each derived step is checked against the clean context (only validated steps propagate forward).

Range: [−1.0, +1.0] · Default weight: 1.0
  • Valid entailment: +1.0
  • Invalid entailment: −1.0
  • Duplicate derivation: −0.5 (new: penalizes reasoning loops)
  • Meta-predicates: subgoal/unreachable/unknown supported
Repetition detection: A derived_logic_set tracks every derived expression. If a step re-derives an already-known fact, it receives −0.5 instead of +1.0 and does not enter the clean context. This directly penalizes the reasoning loops that caused 15 of 22 failures.

📝 Format

Structural compliance with the <think> + <step> generation format. 5 sub-checks, each worth 0.20.

Range: [−1.0, +1.0] · Default weight: 1.0 · Capped at 1.0
  • <think>...</think> wrapper present
  • At least one <step> tag parsed
  • Steps use <text>/<logic> sub-tags
  • Valid logic syntax in each step
  • Answer line present after </think>

✅ Answer UPDATED

Final answer correctness. Supports binary, ternary, numeric, expression, and multi-choice matching.

Range: [−1.0, +1.0] · Default weight: 1.0
  • Correct answer: +1.0
  • Wrong answer: −1.0
  • No answer produced: −0.5 (was 0.0 — looping is no longer free)
  • "Unknown" on binary task: −1.0 (new: answer-type-aware validation)
Answer-type awareness: Each training example now carries an answer_type tag (binary, ternary, ternary_unknown). When the task is binary (yes/no), predicting "unknown" is immediately penalized −1.0. This addresses the dominant failure mode where 67.9% of LogicBench errors were "unknown" predictions on a binary task.

📍 Premise Grounding NEW

Checks that each premise step actually corresponds to a sentence in the input context. Catches hallucinated rules.

Range: [−1.0, +1.0] · Default weight: 0.5
  • Premise matches context sentence (Jaccard ≥ 0.5 or substring): +1.0
  • Premise has no match in context (hallucinated): −1.0
  • No premise steps: 0.0 (neutral)
Why this matters: The model learned to state fabricated rules as type="premise" to bypass SAT verification (premises are unconditionally trusted in the clean context). This includes reversed implications ("A→B" stated as "B→A") and entirely invented rules. The grounding check uses Jaccard token overlap + substring containment against the input context sentences to detect these.

📏 Length Penalty

Multi-factor penalty for degenerate output lengths. Always ≤ 0 (penalizes, never rewards).

Range: (−∞, 0.0] · Weight: built into penalty
  • Too few derived steps (< min_steps): −2.0 × wlp
  • Too many total steps (> max_steps): quadratic penalty
  • Low validity ratio (< 30% valid steps): proportional penalty
  • Truncation detected (~95% of max tokens): −0.5

🚪 Soft Answer Gate UPDATED

Modulates the answer reward based on reasoning quality. Prevents lucky guesses from being rewarded.

Applies to: answer component
  • Valid chain + right answer: full +1.0 credit
  • Any chain + wrong answer: full −1.0 penalty
  • Invalid chain + right answer: partial credit (20–100%)
Soft gating (new): Previously, the gate was binary — invalid reasoning zeroed the answer reward entirely, creating a chicken-and-egg problem for hard theories. Now, a configurable floor (default 20%) ensures some answer signal always flows through:

gate = floor + (1 − floor) × max(0, (consistency + 1) / 2)

consistency = −1.0→gate = 0.20 (floor)
consistency =  0.0→gate = 0.60
consistency = +1.0→gate = 1.00 (full)

Example: How a Reasoning Loop Gets Penalized

Before vs after the reward changes, for a model that re-derives big(lion) 40 times and outputs "unknown" on a binary task.

Before (old reward)

  • Consistency: 40 valid SAT checks → +1.0
  • Format: well-formed steps → +0.8
  • Answer: no answer extracted → 0.0
  • Length: quadratic penalty → −0.7
  • Grounding: did not exist → 0.0
Total ≈ +1.1 (positive reward for looping!)

After (new reward)

  • Consistency: 1st derivation +1.0, 39 duplicates × −0.5 → −0.93
  • Format: well-formed steps → +0.8
  • Answer: no answer → −0.5; "unknown" on binary → −1.0
  • Length: quadratic + truncation → −1.2
  • Grounding: 3 hallucinated premises → −0.3
Total ≈ −2.6 (strong negative signal)
