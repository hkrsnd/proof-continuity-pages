Logic Generalization Strategy

Paper 1 execution plan to improve transfer beyond narrow LogicBench-style training.

Core Message

Generalization is a coverage + objective problem, not only a dataset-size problem. We will scale LogicBench++ with strict symbolic guarantees while increasing language diversity and keeping transfer as the primary evaluation gate.

Typed Composition Strict Verification Naturalized Language Bridge Verifier-Aligned SFT + GRPO

Data Strategy (LogicBench++)

  1. strict_core: exact typed chains with no shortcut proofs.
  2. rich_noise: strict core plus distractor structures.
  3. naturalized_bridge: same logic graphs with benchmark-like NL realizations.
  4. 3-class supervision (True, False, Unknown) with contradiction controls.

Training Strategy

  1. SFT: typed proof traces + final label, balanced by logic type and label.
  2. GRPO: reward on final correctness, step validity, and consistency across paraphrases.
  3. Use KL and anti-shortcut controls to avoid collapse.

Evaluation Gate

  1. Never train on external benchmark test splits.
  2. Track transfer on FOLIO, ProofWriter, and ProntoQA every cycle.
  3. Use ablations to isolate causal gains (data coverage vs language diversity vs reward).
